# Immutable This page exposes a forward-only api. ### Scenario Urls are minted with the `organization` revision. State changes to the app cause an update to the revision of the organization. ### Caching * **Expiration** caching is set to `max-age={oneyear}` * **Validation** caching is not needed (eg `etag`) ### Side effects * Server caching is dead simple * Clients can move forward and backward through revisions of the app * Clients are in control of upgrading when notified of state changes
